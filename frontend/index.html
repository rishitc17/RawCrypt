How it works

The scoring method that keeps every cipher in play.

Cost vs security

Every cipher has a cost (1-10) and a security rating (1-10). Communicators weigh them based on the message's security level — like choosing between a bike lock and a bank vault.

Empirical learning

Agents track which ciphers get broken and which attacks succeed. They update their strategy with exponential moving average — recent results matter more than ancient history.

Softmax sampling

Agents pick actions via a softmax over utilities, so they keep exploring instead of locking onto one option. That's why you'll see 7-9 ciphers in use — not all RSA.

The 9 ciphers

From the trivial Caesar shift to toy RSA — all implemented from scratch in Python.

cost 1 sec 1 Shift

The Caesar cipher: every byte shifted by a fixed amount. Trivial to brute-force.

cost 1 sec 2 Rail Fence

A transposition cipher that writes the message in a zig-zag pattern across N rails.

cost 2 sec 3 Permutation

Each 8-byte block has its bytes rearranged according to a fixed permutation map.

cost 2 sec 3 Vigenère

A polyalphabetic cipher that uses a repeating key — defeated the frequency analysis for 300 years.

cost 3 sec 4 Substitution

Each printable byte maps to a different byte. 95! possible keys, but frequency analysis breaks it.

cost 2 sec 5 Stream

An LFSR generates a pseudo-random keystream, XORed with the plaintext byte-by-byte.

cost 3 sec 6 Feistel

A 4-round toy Feistel network with XOR + nibble-swap. Same circuit encrypts and decrypts.

cost 5 sec 8 AES

A 2-round toy AES with 16-bit blocks. Real AES uses 128-bit blocks and 10-14 rounds.

cost 8 sec 10 RSA

The only asymmetric cipher. Based on the difficulty of factoring n = p × q. Unbreakable in-sim.
